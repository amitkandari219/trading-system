CRITICAL FINDINGS (7 ISSUES)
================================================================================

1. NO CIRCUIT BREAKER IN SIZER
   Location: core/unified_sizer.py lines 211-213
   Risk: Can allocate 72% of capital margin (violates 60% cap)
   Fix: Pass existing_margin to sizer, check total before allocation
   Impact: 🔴 CRITICAL

2. RECURSIVE ROUNDING ERRORS
   Location: core/unified_sizer.py lines 179-207
   Risk: Multipliers of 0.8×0.5×0.6 get rounded to 1 lot at each step
         Instead of 0.24 (24% of base), you get 100% sizing
   Fix: Single round() at final step, use floor() for de-sizing
   Impact: 🔴 CRITICAL

3. NO LIVE POSITION TRACKING
   Location: core/orchestrator.py
   Risk: Tries to open 3rd position when margin exhausted
         No sync with broker's actual position state
   Fix: Query kite_bridge.get_positions() before sizing in live mode
   Impact: 🔴 CRITICAL

4. NO DAILY LOSS HALT
   Location: core/orchestrator.py
   Risk: Config defines DAILY_LOSS_FRACTION=5% (₹50k) but never checked
         Could lose ₹50k and keep trading
   Fix: Check daily P&L before run_daily_session(), return [] if limit breached
   Impact: 🔴 CRITICAL

5. NIFTY EXPIRY NOT SPECIFIED
   Location: core/orchestrator.py line 742
   Risk: Order placed as 'NIFTY' without expiry → defaults to far-month
         Can trade wrong contract (1-5% price difference)
   Fix: Specify 'NIFTY23MAY' format with nearest Friday expiry
   Impact: 🔴 CRITICAL

6. NO STOP-LOSS LINKAGE TO ENTRY
   Location: core/orchestrator.py lines 735-756
   Risk: Entry order placed but no correlated SL order
         Manual SL required or unlimited downside if forgotten
   Fix: Place bracket order with SL trigger_price after entry confirms
   Impact: 🔴 CRITICAL

7. INTRADAY DATA STALE
   Location: core/orchestrator.py lines 517-607
   Risk: Intraday signals (EXPIRY_PIN_FADE) evaluated on 200-day daily closes
         Not real-time 5-minute bars
   Fix: Separate _fetch_intraday_data() method for 5-min ticks
   Impact: 🔴 CRITICAL
